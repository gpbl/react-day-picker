# Styling

<Description>
  Change the appearance of the HTML elements, such as headings, grid cells and
  buttons.
</Description>

---

To work with the default styles, import the CSS file into your app's root component or include it from your HTML page:

```tsx title="App.tsx"
import 'react-day-picker/style.css';
```

From here, you have different options to customize the appearance of the calendar, depending on your project setup.

## CSS Variables

The default styles use CSS variables to make it easier to customize the base appearance of the calendar. You can override the CSS variables to change the appearance of the calendar.

```css title="global.css"
:root {
  --rdp-accent-color: indigo;
  --rdp-background-color: gray;
}
```

## Overriding the default style

To further customize the styles, override the DayPicker CSS. For example, to change the background color of the day cells, add this style sheet to your `global.css` or `app.css`.

```css title="global.css"
/* Change the background color of the day cell. */
.rdp-day {
  background-color: #f0f0f0;
}
```

## CSS Modules

If you are using CSS modules to generate unique class names, import from `react-day-picker/style.module.css`
instead. Then, use the `classNames` prop to pass the custom class names to DayPicker.

```tsx
import { DayPicker } from 'react-day-picker';
import classNames from 'react-day-picker/style.module.css';
// ...
<DayPicker classNames={classNames} />;
```

## Inline Styles

To change the appearance of any DayPicker element via inline-styles use the `styles` prop.

Using inline styles won't override the default class names. Mouse hover effects cannot be styled inline.

```tsx
const monthCaptionStyle = {
  borderBottom: '1px solid currentColor',
  paddingBottom: '0.5em'
};
// ...
<DayPicker
  styles={{
    month_caption: monthCaptionStyle
  }}
/>;
```
